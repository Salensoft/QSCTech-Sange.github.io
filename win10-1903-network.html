解决Windows10 1903更新的KB4515384补丁造成的无法联网的问题
发表于 2019-10-17| 更新于 2019-10-17|网络
字数总计:660|阅读时长: 2 分钟
| 阅读量:|评论数:

背景

今天被老师喊去修她办公室校网连接不上，具体情况是校网VPN客户端提示“未知错误，请检查连接拨号连接”，遂给信息中心打电话。信息中心的人说未知错误这个问题只能重装电脑，我这暴脾气就上来了，你咋不说重买电脑呢？

查了下log，研究了一下电脑的设置，发现问题根源确实不在校网上（属实难得）。IPv4手动设置得很正确，也连得上内网。手动建VPN连接和使用校网VPN客户端都不能连接上，着实让我有点黔驴技穷。

近日，Windows10推送了KB4515384补丁，该补丁与部分型号的网卡适配器产生了冲突，有线和Wifi都无法使用。当然我一开始是不知道这是由于这个补丁引起的。

如果你发现更新系统后右下角任务栏没有网络的图标，设置里的网络状态里是一条直线，那么你遇到了同样的问题了。

题外话，如果你搜索这个补丁的名字的话，你会发现它造成了很多很多的问题……从音频问题、PIN 登陆问题，到 CPU 占用率高、网络适配器损坏、误删开始主菜单……

解决方法

两个方法。

  1. 当然是听信息中心的话重装啦。开玩笑的，基础办法是卸载更新回滚到上一个系统。当然这个可能又一次更新然后又炸了……
  2. 修改注册表。新建一个记事本，里面输入
Code

然后将其保存成xxx.reg的形式，前面名字随便起，后缀名要确定是.reg。双击运行它，接着重启即可。看看，是不是网络图标又回来啦？这个方法的灵感要特别感谢cc98，还记得前面说的内网能进吗？既然能上内网而且又想不到解决方法，当然要先上cc98消磨时间啦~
